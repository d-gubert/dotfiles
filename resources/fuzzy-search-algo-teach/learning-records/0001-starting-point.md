# Starting point: wants to understand fuzzy search design decisions

User wants to learn the fuzzy search algorithm from first principles, with emphasis on understanding *why* each design decision was made and what specific problem it solves. They asked for a curriculum broken into independent, increasingly complex tasks.

**Implications**: Frame every lesson around a problem → solution structure. Do not just teach mechanics — always show the design decision and the problem it addresses. The user is engineering-minded and wants to understand the algorithm deeply enough to read real implementations without confusion.
